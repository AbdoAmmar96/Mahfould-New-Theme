## 2) المبادئ المعمارية الخمسة

كل قرار في النظام راجع لواحد من دول. لو حد في الفريق هيضيف حاجة، لازم يعدّي عليهم الأول.

### ① محرك واحد، أقسام لا نهائية
الحجز والإتاحة والتقييم والمفضلة كلهم **polymorphic**. إضافة قسم جديد = **جدولين + trait**، وصفر تعديل في المحرك.

الدليل العملي: التجارب والكروز اتضافوا بعد ما المحرك خلص، ومحدش لمس فيه سطر.

```php
class ExperienceSession extends Model
{
    use HasAvailability;                 // ← ده كل اللي محتاجه

    public function isPooled(): bool      { return true; }
    public function inventoryCount(): int { return $this->capacity; }
}
```

### ② دفتر، مش عدّاد
`availabilities` **مفيهوش** عمود `capacity_booked`. العدد المحجوز بيتحسب من `booking_items` مباشرةً.

**السبب:** العدّاد المنفصل بيقع في drift مع الحجوزات الفعلية. ودي أشهر بق في أنظمة الحجز — وبتكتشفه لما يبقى متأخر وعندك حجزين على نفس الغرفة.

الثمن: استعلام COUNT بدل قراءة عمود. مع الـ index الموجود ده مش عبء.

### ③ ثلاث طبقات قفل
```
1. Cache::lock (Redis)     ← يمنع طلبين متزامنين يدخلوا الحساب سوا
2. transaction + recheck   ← الحالة اتغيّرت وإحنا مستنيين؟
3. UNIQUE index في MySQL   ← خط الدفاع الأخير
```
الأولى بتمنع 99% من التزاحم. التالتة بتضمن إنه **مستحيل** يحصل حجز مزدوج حتى لو الأولى والتانية اتكسروا.

**اتجرّب فعلياً:** 13 حالة اختبار على MySQL — حجز مزدوج اترفض، إعادة حجز بعد إلغاء عدّت، 3 نسخ من نوع غرفة عدّوا والرابعة اترفضت.

### ④ البيانات محايدة المصدر
`partner_rates.source` بياخد `manual` أو `api` أو `feed`. الاستعلامات والخدمة والواجهة **مش بيفرق معاهم**.

يعني تبني المقارنة النهارده ببيانات وهمية، ويوم ما تربط بوكينج.كوم بتضيف صفوف بـ `source='api'` — **ولا سطر كود بيتغيّر**.

### ⑤ الواجهة مدفوعة بالإعدادات
كِت القوايم بياخد الفلاتر كـ config. 8 أقسام = **ملف واحد** بـ 8 تكوينات، مش 8 نسخ.

```jsx
const GROUPS = [
    { key: 'stars',     label: 'عدد النجوم',   type: 'stars' },
    { key: 'meal_plan', label: 'نظام الوجبات', type: 'check', options: [...] },
];
<FilterPanel groups={GROUPS} value={f} onChange={apply} />
```

| الرحلات | ميعاد قيام | مجمّع | `tour_departures` 🔜 |
| **الفنادق** | نوع غرفة × الكمية | مجمّع | `room_types` ✅ |
| **المطاعم** | ترابيزة × فترة | حصري | `restaurant_tables` ✅ |
| **التجارب** | جلسة بعدد أماكن | مجمّع | `experience_sessions` ✅ |
| **الكروز** | نوع تذكرة/كابينة | مجمّع | `cruise_ticket_types` ✅ |
| السيارات | العربية نفسها | حصري | `cars` 🔜 |
| الباصات | الباص (تأجير) / المقعد (خط) | حصري | `buses` · `bus_seats` 🔜 |
| صاحب السعادة | باقة بسعة يومية | مجمّع | `sahb_packages` 🔜 |

### الفرق بين الحصري والمجمّع

```
حصري (ترابيزة، مقعد، عربية)      مجمّع (نوع غرفة عنده 10 غرف)
├── نسخة واحدة                    ├── 10 نسخ
├── unit_index = 0 دايماً         ├── unit_index = 0..9
└── مين ياخدها أول = خلاص         └── الحجز بياخد أول رقم فاضي
```

العمود `unit_index` هو اللي بيخلي القيد الواحد يخدم الاتنين. من غيره الـ unique index كان هيمنع حجز غرفتين من نفس النوع في نفس الليلة بالغلط.

## 4) محرك الإتاحة — القلب

### الجداول الثلاثة

```
availabilities          تجاوزات التقويم — سطر بس لو فيه استثناء
├── availableable (morph)
├── date · start_time                     ('00:00:00' = اليوم كله)
├── capacity_override · price_override
├── is_closed
└── UNIQUE (type, id, date, start_time)

booking_items           دفتر المخزون — أهم جدول في النظام
├── booking_id (nullable) · hold_token (nullable)
├── unit (morph) · unit_index
├── date · start_time · end_time
├── status: held | booked
├── expires_at · released_at
├── active_flag  ← عمود مولّد: 1 لو نشط، NULL لو اتحرّر
└── UNIQUE (unit_type, unit_id, unit_index, date, start_time, active_flag)

booking_holds           "السلة" — سياق الجلسة المؤقتة
├── token · user_id / session_id
├── bookable (morph) · start_date · guests
├── estimated_total
└── expires_at · converted_at
```

### مسار الحجز

```
  العميل يختار
       │
       ▼
  ┌─────────────────┐
  │ Cache::lock     │  Redis، 5 ثواني انتظار
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │ transaction     │
  │  ├─ recheck     │  remainingOn() — الحالة اتغيّرت؟
  │  ├─ nextIndex() │  أول نسخة فاضية
  │  └─ INSERT      │  booking_items status=held
  └────────┬────────┘
           ▼
  booking_holds  (15 دقيقة — 10 للمطاعم، 20 للفنادق)
           │
     ┌─────┴─────┐
     ▼           ▼
   دفع ناجح    انتهت المهلة
     │           │
     ▼           ▼
 status=booked  holds:expire → released_at = now()
 hold_token=null              الوحدة رجعت متاحة
```

### التفصيلة اللي بتوقّع الناس

الـ unique index على `start_time` **بالظبط**. يعني ترابيزة محجوزة 20:00 وجلستها ساعتين، القيد **مش هيمنع** حد يحجزها 21:00.

**التداخل بيتمسك في الاستعلام**، مش في القيد:

```php
// HasAvailability::bookedCountOn()
$q->where('start_time', '<', $endTime)
  ->where(fn ($w) => $w->whereNull('end_time')->orWhere('end_time', '>', $startTime));
```

⚠️ **بشرط إن `end_time` تكون متعبّية.** لو سيبتها فاضية للمطاعم أو الباصات، الحماية دي بتضيع بالكامل.

### الحجوزات الممتدة

3 ليالي × غرفتين = **6 سطور** في `booking_items`، مش سطرين. سطر لكل يوم لكل نسخة.

ده اللي بيخلي استعلام الإتاحة اليومي موحّد لكل الأقسام — الفندق والعربية والباص كلهم نفس الاستعلام.

## 5) طبقة الثقة

الميزة التنافسية الحقيقية. أي حد يقدر يعمل منصة حجز؛ سجل ثقة مربوط بترخيص حقيقي أصعب بكتير.

```
companies
├── name · logo · about · founded_year
├── license_no · license_authority (ETAA) · license_expires_at
├── verification_status: unverified | pending | verified | suspended | expired
├── rating · completed_bookings · response_rate · response_time
└── commission_rate (override)

company_documents    سجل تجاري · بطاقة ضريبية · ترخيص سياحي
company_user         pivot بـ role: owner | manager | staff
```

| الحالة | يقدر يبيع؟ |
|---|---|
| `verified` | ✅ |
| `pending` · `unverified` | ❌ عرض في الدليل بس |
| `expired` | ❌ — job يومي بيقلبها تلقائياً |

**قرار تجاري معلّق:** هل التوثيق شرط إجباري للبيع؟ الصرامة بتبني الثقة بس بتبطّأ نمو العرض في أول 6 شهور. القرار ده لازم يتاخد **قبل** تسجيل أول شركة، لأن الرجوع فيه بعدين بيحرق مصداقية الشارة.

## 6) طبقة المقارنة

```
partners        بوكينج.كوم · أجودا · مصر تورز · شركات محلية
partner_rates
├── partner_id · rateable (morph) · room_type_id · date
├── price · currency · url · is_available
└── source: manual | api | feed        ← المفتاح
    fetched_at
```

`BestPriceService::compare()` بيرجّع سعرك جنب أسعار الشركاء مرتّبين، ومعاهم `savings` و`is_cheapest`.

⚠️ **مكوّن `PriceComparison` عنده prop اسمه `guaranteed` قيمته `false`.**
لما تبقى `false` بيكتب "مقارنة الأسعار" و"الأسعار استرشادية". لما تبقى `true` بيكتب **"أفضل سعر مضمون"** — ودي التزام تعويض قانوني لو العميل لقى أرخص. سيبها `false` لحد ما يبقى فيه سياسة مكتوبة وميزانية.

## 15) الملخص لمن معاه دقيقة

**اللي اتبنى صح:** محرك إتاحة polymorphic بثلاث طبقات حماية، متجرّب على قاعدة بيانات حقيقية، بيستوعب أي قسم جديد من غير تعديل فيه. ومسار حجز شغّال من الاختيار للتأكيد.

**اللي لسه:** طبقة الثقة (الميزة التنافسية الأساسية)، و18 صفحة فرونت، والبنية التحتية.

**أكبر خطر:** إن التصميم بيوعد بمنتج أكبر بكتير من اللي فيه قدرة تنفيذية — 8 أقسام ومنصة سواقين ومقارنة ومحفظة وكول سنتر 24/7. **الإطلاق بـ 3 أقسام متقنة أفضل من 8 نصّهم شغّال.**

**أول خطوة:** شغّل الكود على سيرفر حقيقي. لسه ولا سطر منه اتشغّل.
